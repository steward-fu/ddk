Installing the Windows Me DDK

This topic describes how to install the Windows Me DDK. The following notation conventions are used.

Convention

Description

 

%NTDDK%

Root directory of the installed Windows Me DDK

%MSVCDIR%

Root directory of the installed Microsoft Visual C++ development environment.

 

System Requirements

The minimum requirements for using the DDK include the following software and hardware:

Microsoft Windows Me operating system final release

Microsoft Visual C++ 6.0 (including service packs) for x86-based systems.

Important

You must install the Visual C++ compiler and development environment before running winmebld.bat. Otherwise, the winmebld.bat file in the Windows Me DDK will not set up the correct environment when you run it. For more information, see Setting Up and Using the Windows Me Driver Build Environment.

 

CD-ROM drive, if installing from CD or testing CD drivers.

32MB RAM.

193 MB hard disk space for the full installation (35 MB of hard disk space for a minimum installation, which does not include the sample driver source code, Help files, or other documentation files).

Legacy driver samples. The legacy driver samples require a 16-bit compiler and linker. The 16-bit compiler and linker are provided in the %NTDDK%\bin\win_me\bin16 directory.

 

Using the Setup Program to Install the DDK

1. Run Setup.exe from the Windows Me DDK CD-ROM, or download the self-extracting executable file, which will begin Setup.exe upon completion of the uncompression. Setup.exe is a Windows-based application you can run from either the Start or Run button, or from an MS-DOS prompt.

2. Click Next to install the DDK.

3.Read the Software License Agreement and, if you agree with its terms, click I accept the Agreement and click Next.

4.Select the drive and directory where you want to install the Windows Me DDK and then click Next.

 

Note

The Windows Me DDK cannot be installed in a directory that does not follow the 8.3 naming convention.

 

5. Select the component groups you want to install. Click Details to select or deselect subcomponents. Click Next.

6. Verify that the appropriate components will be installed and click Finish.

7. After the installation of components is completed, click Next and then click Exit.

 

Installation Notes

The Setup program installs the selected Windows Me DDK components in the specified installation directory and creates program folders.

You should increase your environment space because the Windows Me DDK, compiler, and assembler all use additional environment space for storage of search paths.

Note:

This is done automatically if you use a Build Environment short cut the from the Development Kits\Windows Me DDK group on the Start Menu.

 

 

Note

The Windows Me DDK uses the current VCVars32.Bat as part of creating the build environment. This includes setting your current path to include the installation source of Visual C++.

 

Manually increasing the environment space if you do not use a Start Menu shortcut

Create a shortcut for the MS-DOS Prompt and place the following in the Cmd line of the Program Tab.

%windir%\command.com /e:4096

 

This command gives you an environment of 4,096 bytes. If you receive "out of environment space" messages, you should increase the size of the environment further.

 

Setting Up and Using the Windows Me Driver Build Environment

This topic provides instructions for establishing a WDM driver build environment and using the Build utility to build drivers.

Using WINMEBLD.BAT to Set Up the Driver Build Environment

A program folder named Development Kits\Windows Me DDK is created during the DDK installation. This folder contains items for automatically creating the build environments. They are:

  • Free Build Environment for Win Me
  • Checked Build Environment for Win Me

Each time you restart the operating system, and before you build a driver, click one or the other of these program folder items. These items call winmebld.bat in %NTDDK%\bin\win_me\bin to create the driver build environment using the correct environment variables.

Running winmebld.bat manually

Use the following syntax at the MS-DOS prompt or in the Start or Run box:

winmebld%NTDDK% [free | checked]

 

For example, from the C:\NTddk\bin\win_me\bin> prompt, type winmebld C:\NTddk free

The first argument, <destination>, specifies the directory where the DDK is installed. Note that in the case of default setup, the destination is \NTddk. The optional second argument specifies the target build environment; the default type is free.

Verifying DDK Installation

Start either the free or checked build environment by the choosing the appropriate icon from the Windows Me DDK menu. Verify your installation of the Windows Me DDK by running nmake from the %NTDDK%\src\win_me\general directory. You may also want to verify other projects by running nmake in the base, block, display, and storage\lockcd directories under the %NTDDK%\src\win_me location.

 

Starting to Build Your Own Drivers

Once you have built the sample drivers to verify the Windows Me DDK installation, you can start building your own driver. The Windows Me DDK samples use the VC nmake utility to build its components.

Note: The Windows 2000 DDK uses the build utility to build drivers. If you want to integrate Windows Me samples to use the build utility, you need to create your own DIRS/SOURCES files for your project. There are several ways to do this, but the easiest is to create a sources file which builds no target. It can then call your existing build process. In this case, the build utility is used strictly to spawn nmake. The below example is a dummy sources file: 


======== SOURCES file ========

TARGETNAME=
TARGETTYPE=NOTARGET
TARGETPATH=
SOURCES=

For more detailed information about the build utility and the Windows 2000 DDK build environment, see the Driver Writer's Guide in the Windows 2000 DDK Online Help. For the demostration of the NMAKE/BUILD integration, see the sample in %NTDDK%\src\win_me\general directory.

For building WDM drivers, refer to the Windows 2000 DDK documentation. Note that much of the detailed information about building kernel-mode drivers in the Windows NT 4.0 and Windows 2000 DDKs also applies to building WDM drivers.

 

Debugging Windows Me

The debugger tool provided in the Windows Me DDK is the Windows System Debugger debugger.exe. For detailed information on how to install and use debugger.exe, see "Windows Me Debugging and Other Tools\Windows 9x Debugging" under Millennium DDK in the online documentation

A remote terminal is provided for additional debugging capabilities. RTerm, located in the \bin\win_me\fre and \bin\win_me\chk folders of the DDK, supports source-level debugging as well as being Win32-aware. The new application handles higher transfer rates, up to 115.2 kbps, the Pentium instruction set, and the ability to load debug symbols dynamically from RTerm. Additional information is available under the application's Help menu or by typing -? or .-?, using the debugger's command line switches.

The Windows Me DDK RTerm may be used in combination with the DDK debugger, debugger.exe, but is incompatible with the older WDEB applications. The previous Rterm utility, Rterm98, must be paired with WDEB98 to function. Likewise, the older Windows 95 version of RTerm, or an equivalent remote terminal application such as HyperTerm, may be used with WDEB386.EXE.

Release Notes

Internet Explorer Required for Documentation

This refresh of the Windows Me DDK requires Microsoft Internet Explorer 4.01 or later for full functionality. If the setup program does not detect IE 4.01 or later on your hard drive, the documentation will not be installed. The latest version of Microsoft Internet Explorer can be downloaded separately from the Web at http://www.microsoft.com/ie.

Running Windows Me DDK tools on Windows 2000 and Windows XP

The Windows Me DDK tools must be run on a computer under Windows Me. The tools will not necessarily run under Windows 2000 or Windows XP (depending on the tool).

Visual C++ 6.0 Service Pack 3 Recommended

Visual C++ 6.0 Professional Service Pack 3 (or greater) is required to build samples.
