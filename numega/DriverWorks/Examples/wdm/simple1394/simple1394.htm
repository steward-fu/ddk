Simple1394 Sample

SUMMARY

The simple1394 sample illustrates how to write the simplest 1394 driver. The driver performs asynchronous read and write operations to the device’s address space.  It also maintains a copy of the bus reset generation count and overrides KPnpDevice::OnBusReset to update the generation count when a bus reset occurs.

The sample includes the driver smpl1394.sys, and a test application, test_simple1394.exe, that sends read and write requests to the driver.

This sample may be tested with any 1394 device.  It will be necessary to put the device’s hardware ID in the INF file, and to set the offset in the device’s address space to which the driver reads and writes

 

Note:  The simple1394 driver cannot be run on each of two PCs connected with a 1394 cable.  In order to read and write to another PC, the driver on the other PC must allocate an address range in order to grant smpl1394.sys access.

 

BUILDING THE SAMPLE

The driver, smpl1394.sys, and application, test_simple1394.exe, may be built from the command line or from within the Visual C++ IDE.    To build from the Visual Studio IDE, open the workspace file, simple1394.dsw.  From the build menu, select the configuration to build, and then select “Build” or “Rebuild All” to build that configuration.  Both free and checked builds are available.

To build from the command line, start a command prompt with the DDK checked or free build configuration.  Navigate to the Simple1394 directory and type “build”.  The environment variable DRIVERWORKS must be set to the DriverWorks directory.

Note: The DriverWorks library, vdw_wdm.lib, must be built before smpl1394.sys may be built.

 

RUNNING THE SAMPLE

The driver, smpl1394.sys, must be loaded through the device manager.  If the 1394 device is already listed in the device manager, double click on the device, and, when the properties dialog comes up, select Update driver from the driver tab.  Choose to specify the location of the driver file, and browse to the folder containing simple1394.inf.  The device manager should then load the driver.

If the device is not in the device manager, the driver may be loaded by using the Add New Hardware option in the control panel.

On Windows 2000 and XP, it is also possible to load the driver using the EZDriverInstaller available in the start menu under NuMega DriverStudio | Tools.

Note: To install smpl1394.sys on Windows XP, the EZDriverInstaller must be used.  Windows XP automatically loads a network driver when another PC is connected through 1394.  Because the smpl1394 driver is not in the same device class as the network driver, Windows will not allow the driver to be installed through the device manager in the normal way.
