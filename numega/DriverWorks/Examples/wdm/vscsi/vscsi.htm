Virtual Scsi Port Driver

[This is preliminary documentation and subject to change.]

Summary

The Virtual Scsi Port Driver sample demonstrates how to write a bus driver using DriverWorks framework. The sample includes a scsi port driver for a bus of network disk servers.

Theory of Operation

On windows 2000/XP,.NET platform scsi port drivers serve as bus drivers for storage type of devices on a particular bus. For example, atapi.sys is a scsi port driver for IDE bus. It enumerates all IDE devices and translates SCSI commands into native IDE commands. Our sample virtual scsi port driver is the same kind of bus driver only for a different network based bus of disk servers. To enumerate all devices on that bus vscsi driver broadcasts special udp packets and collects the response packets. To communicate with a particular disk server vscsi opens a TCP connection and sends read/write commands over TCP protocol.

A Disk server is implemented in iserver driver. A scsi port driver is implemented in vscsi.sys driver.

Building the Driver

First, build the DriverWorks libraries. Next build the vscsi and iserver projects using either MSVC or DDK build.exe.

Installing the Driver

First install a iserver driver using server.inf file and then install vscsi port driver using vscsi.inf file on another machine, which is connected over the network to the first one.
