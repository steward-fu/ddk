1.1  Firefly: DriverWorks Sample Driver

SUMMARY

The Firefly sample is an Windows Driver Model (WDM) upper filter driver for the Microsoft USB Intellimouse Wheel written using the DriverWorks C++ Framework.  The driver is an HID upper filter driver for the HID mouse driver.  The driver can perform I/O with the HID stack to turn on and off the tail light on the USB mouse.  The sample is based on the Firefly sample from the Windows DDK.  This sample replaces the driver from the DDK Firefly sample and can be used with the sample application from the DDK.

 

PROGRAMMING TOUR

Firefly driver class, Firefly, derives from KDriver and implements DriverEntry() and AddDevice() methods, which performs driver's initialization and device object creation (FireflyDevice class instantiation.)  The code was generated using the DriverWorks Driver Wizard.

The FireflyDevice class derives from KWdmFilterDevice to act as a WDM filter driver.  KWdmFilterDevice handles all of the plug and play and power management tasks and also the default "pass-thru" behavior of the driver.  The FireflyDevice class provides a Windows Management Instrumentation (WMI) interface for applications to control the USB mouse tail light.  Specifically, the class has data member m_TailLight of type KWmiDataBlock<MOUSE_LIGHT> that exposes a WMI boolean Data Item representing the state of the tail light on the mouse (on or off.)  In response to WMI request to set the data item (tail light on or off,) the driver issues requests to the HID stack to turn the tail light on or off.  By default, the DriverWorks framework handles updating the data members of the WMI data block in response to the set item request.  For this driver, special behavior is required to issue requests to the HID stack.  So C++ template method KWmiDataBlock<MOUSE_LIGHT>::Set (located in file FireflyDevice.cpp) is specialized to handle this.  The method FireflyDevice::SetFeature is used to issue the request.  This method opens the HID collection using ZwCreateFile and calls IOCTL_HID_GET_COLLECTION_INFORMATION & IOCTL_HID_GET_COLLECTION_DESCRIPTOR to get the HID preparsed data. It then calls HidP_GetCaps using the HID preparsed data to get the HID capabilities of the device. After getting the HID capabilities of the device, the driver creates an HID feature report to set or clear the feature that causes the light to turn on or off.

See the Firefly sample in the DDK for additional info.  Also see complete HID documentation in the DDK documentation.

 

BUILDING and INSTALLING THE SAMPLE

The sample requires Windows XP or later DDK.

Start Microsoft Visual Studio .NET IDE. Open driver project file, Firefly.sln, and build the driver either using the standard IDE build buttons or DriverStudio DDK Build button.  Be sure to select a DDK to build with using the DDK Build Settings dialog from the DriverStudio menu.  Alternatively, one can build the sample by typing build -ceZ from the command prompt of a DDK build environment.  A successful build produces Firefly.sys.

To install the driver, first install the USB Mouse hardware and verify that the mouse works correctly.  Then, using Device Manager, under "Mice and other pointing devices" update the USB mouse driver.  Select the Firefly.inf file for installation.

RUNNING THE SAMPLE

Once installation is complete, the driver can be exercised using the FireflyTest.htm file.  Open the Fireflytest.htm file in the Internet Explorer Browser.  When the webpage loads, the driver's WMI information is accessed.  Use the buttons to turn the taillight on, off, start flashing, and stop flashing.

The driver can also be exercised using test applications from the DDK for the Firefly sample.  Use Flicker.exe to flash the light.  Refer to the Firefly DDK sample for more information.

The sample runs on Windows 2000 and Windows XP and up.  Windows 9x is not supported.

TOOLS

Use Compuware SoftICE or Visual SoftICE debugger and DriverMonitor trace viewer tool to further exploit the implementation details of the sample.
