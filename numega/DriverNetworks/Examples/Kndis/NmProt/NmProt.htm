NmProt NDIS Sample Protocol Driver (5.1)

Summary

This directory contains NM Protocol Driver project files for the protocol sample driver called NmProt. This sample provides the same functionality as the ndisuio sample driver in Windows XP DDK, and ndisprot sample driver in Windows Server 2003 DDK. A test application provided with the Windows XP/2003 DDK, named uiotest.exe must be built in order to exercise the driver.

Note:  This NmProt will NOT work with Windows 2000 DDK packapp.exe. For Windows 2000 DDK, see NmPacket sample.

Operation

First, the driver dynamically binds to the selected real and virtual adapters. NmProtBinding::Open() creates a OpenContext for each adapter. The uiotest.exe application uses CreateFile() to access the driver by device name "ndisprot".

The NmProtDevice is a KDevice object that allows the driver to properly handle Irps. The functionality is provided in the NmProtDevice files. It uses an NDIS interface to communicate with any one of the NIC drivers.

When uiotest.exe starts, it issues a IOCTL_NDISPROT_OPEN_DEVICE with specified binding name to get OpenContext saved in IRP FileObject. With a successful return, sequent read/write IRP use the same OpenContext to talk to NDIS adapter. This uiotest.exe also shows how to request/send OID requests to the adapter via DeviceIoControl.

When this sample is up and running, the singleton NmProtDevice should have N instance of NmProtBinding where each instance represents each binding to miniport adapter. NmOpenContext has 1:1 relationship with NmProtBinding for each user mode app.

The driver also demonstrates how to handle dynamic binding and unbinding of adapters (NDIS 5.0 and later). The NmBindingPnp file illustrates how to manage PnP and how Power Management queries are handled. When built for NDIS 5.1, the driver also demonstrates send packet cancel techniques.

Note:  The NM Packet Protocol Driver is compliant with 5.0 and 5.1, both 32-bit and 64-bit platforms. Only the Ethernet adapter is supported.

Before building the driver you should be DriverWorks and DriverNetworks libraries as described in the help files elsewhere.

Then select the desired NmProt configuration and build it in Visual Studio IDE. The drivers will be placed in the appropriate subfolder under …\NmProt\objchk\i386\.

Installing the Driver

On Windows XP and Windows Server 2003, the driver is installed as a Network Protocol Component Type. From the Local Area Connection Properties dialog, select the Install… button. Select the Protocol type of network component to install and click the Add… button. This launches the Select Network Protocol dialog. Click the Have Disk… button and browse to the NmProt.inf file, which is the .inf file for NmProt driver. Select "DriverNetworks Protocol Driver Sample".

At this point the NmProt protocol driver should be installed.

NOTE that the driver does not start automatically. To start the driver either issue >net start ndisprot from command prompt, or reboot the system.

NOTE that for compatibility with the DDK test application, uiotest.exe, the driver’s registered name is “ndisprot”, which is different from its file name, nmprot.sys.

Tools

BoundsChecker for Drivers and SoftICE can be used to test some of the features of this driver. Specifically, the SoftICE “packet” command is useful for displaying and decoding NDIS packet buffers.
