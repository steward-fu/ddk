NmPacket NDIS Sample Protocol Driver

(NDIS 4.0, 5.0, 5.1)

Summary

This directory contains NM Packet Protocol Driver project files for the Packet sample driver called NmPacket . This sample provides the same functionality as the Packet sample driver provided with the Windows 2000 DDK. A test application provided with the Windows 2000 DDK, named packapp.exe  must be built in order to exercise the driver.

Note:  This NmPacket will only work with Windows 2000 DDK packapp.exe. For Windows XP DDK and Windows 2003 DDK uiotest.exe, see NmProt sample.

Operation

First, the driver dynamically binds to the selected real and virtual adapters. Then it creates a deviceobject for each adapter. The packapp.exe sample application uses a symbolic link to access the driver.

The upper-edge interface for the driver is a Kdevice object that allows the driver to properly handle Irps. The functionality is provided in the NmPacketControlDevice files.

At the lower edge, it uses an NDIS interface to communicate with any one of the NIC drivers. Packapp.exe shows how to open the adapter, send or receive packets, and send OID requests to the adapter directly. The sample application loads the driver when it is executed and unloads the driver when it is closed.

The driver also demonstrates how to handle dynamic binding and unbinding of adapters (NDIS 5.0 and later). The NmBindingPnp file illustrates how to manage PnP and how Power Management queries are handled. When built for NDIS 5.1, the driver also demonstrates send packet cancel techniques.

Note:  The NM Packet Protocol Driver is compliant with NDIS 4.0, 5.0 and 5.1 (32-bit only). Only the Ethernet adapter is supported.

Before building the driver you should be DriverWorks and DriverNetworks libraries as described in the help files elsewhere.

Then select the desired NmPacket configuration and build it in Visual Studio IDE. The drivers will be placed in the appropriate subfolder under …\NMPACKET\objchk\i386\.

Installing the Driver

On Windows 2000 and Windows XP, the driver is installed as a Network Protocol Component Type. From the Local Area Connection Properties dialog, select the Install… button. Select the Protocol type of network component to install and click the Add… button. This launches the Select Network Protocol dialog. Click the Have Disk… button and browse to the NmPacket.inf file, which is the .inf file for NmPacket driver. Select "DriverNetworks Packet Driver Sample".

For WinNT 4.0 an OEMSETUP.INF file is provided to simplify the installation process. Right-click the desktop Network Neighborhood icon. Then, select Properties. The network configuration application will run. On the network configuration application, select the Protocols tab. The network Protocols configuration page will be displayed. Select the Add button. Finally, the “Select Network Protocol” window will be displayed. At this point you are ready to install the protocol. Select the Have Disk option and perform the installation.

At this point the NmPacket protocol driver should be installed.

NOTE that the driver does not start automatically. To start the driver either issue >net start packet from command prompt, or reboot the system.

NOTE that for compatibility with the DDK test application, packapp.exe, the driver’s registered name is “packet”, which is different from its file name, nmpacket.sys.

Tools

BoundsChecker for Drivers and SoftICE can be used to test some of the features of this driver. Specifically, the SoftICE “packet” command is useful for displaying and decoding NDIS packet buffers.
