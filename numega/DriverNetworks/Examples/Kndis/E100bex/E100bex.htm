E100bex NDIS Miniport Driver

 

Summary

This directory contains the NDIS Miniport Driver project files for the Intel EtherExpress PRO PCI Ethernet Adapter network card. This sample demonstrates the basic structure of an NDIS 5 deserialized NDIS miniport driver written using the DriverNetworks C++ framework. This is a fully functional NDIS miniport driver tested on both the Windows 2000 and Windows XP systems. The driver is functionally similar to the E100Bex sample driver found in the Windows XP DDK.

The driver demonstrates usage of the following classes:

KNdisMiniAdapter

KNdisIoRange

KNdisMemoryRange

KNdisInterrupt

KNdisPacketList

KNdisSharedReceiveArea

KNdisSharedMemory

KNdisStats

KNdisPacket

KNdisBuffer

KNdisPciInformation

Power Management

 

Building the Driver

Open E100bex.dsp in MSVC 6.0 IDE, select the Win32 NDIS 5 Checked configuration and build it. It creates files E100bex.sys and E100bex.nms. You will need DriverNetworks and the Microsoft Windows DDK installed to successfully build the driver.

Tools

BoundsChecker for Drivers and SoftICE can be used to test some of the features of this driver.

ProgrammingTour

  1. NDIS 4: SendPacketsHandler; multiple packets per send request are handled (E100bexAdapter::SendPackets() in E100bexAdap.cpp).
  2. NDIS 4: NdisMIndicateReceivePacket(); multiple packets per receive indication (E100bexAdapter::ProcessRXInterrupt() in E100bexAdap.cpp).
  3. NDIS 4: Receive buffer grow and shrink; When the adapter driver begins to run out of receive memory, KNdisSharedReceiveArea::Complete() calls _RxAreaAsyncMgr NdisMAllocateSharedMemoryAsyn() to allocate more shared memory for receive buffers (E100bexAdapter::ProcessRXInterrupt() in E100bexAdap.cpp). When the traffic slows down, the driver calls KNdisSharedReceiveArea::Reclaim() to frees the memory (E100bexAdapter::ReturnPacket() in E100Adap.cpp)
  4. NDIS 4: NdisMRegisterAdapterShutdownHandler() is called by DriverNetworks library at initialization time to make sure the adapter has a function that stops generating interrupts and receiving packets into main memory (E100bexAdapter::Initiliaze() and E100bexAdapter::Shutdown() in E100bexAdap.cpp).
  5. NDIS 4: Media disconnect/connect indications are supported. This allows the driver to tell NDIS when the adapter has lost or regained link (E100bexAdapter::CheckForHang() in E100bexAdap.cpp).
  6. NDIS 5: This driver is a de-serialized miniport, therefore it protects its resources using its own spinlocks rather than relying on Ndis to serialize the calls to the driver. This driver only implements a simple method of using spin lock: it has a KNdisSpinLock::Lock() at every entry point and a KNdisSpinLock ::Unlock()  at the return from those entry points. The locks could be done more efficiently in a production driver by moving the locking closer to the resources in the driver that need to be accessed with atomic operations.
  7. NDIS 5: NdisMInitializeScatterGatherDma; this driver always attempts to use scatter gather which is available in NDIS 5 and later.
  8. NDIS 5.1: Send packet cancellation. The CancelSendPacketsHandler allows protocols to cancel pending send packets which are queued in the miniport send queue (E100bexAdapter::CancelSendPackets() in E100bexAdap.cpp).
  9. NDIS 5.1: PNP event notification handler. The miniport drivers can have a PnPEventNotifyHandler to process anything necessary when a PNP event happens (E100bexAdapter::PnPEventNotify() in E100bexAdap.cpp).
  10. NDIS 5.1: AdapterShutdownHandler is now a standard miniport handler starting NDIS 5.1. It eliminates the need to call NdisMRegisterAdapterShutdownhandler (E100bexAdapter::Shutdown() in E100bexAdap.cpp).
